1. Computation for an N-Body Particle Simulation using MPI for parallelized computations

  For my parallel computing class, I built a program that randomly generates light, medium and heavy particles. The program provides each particle with an initial velocity, a mass and an intial position. This is done in a 3 dimensional plane (x, y, and z plane). The particles interact with each other by evaluating Newton's equations of motion for the forces of all other particles that are acting on each individual particles. Using the matrix of force interactions on each particle and Euler's method, I am able to calculate the next velocity step and the next position step in the sequence. Using command line inputs, the user can choose the amount of all 3 types of particles, the time between steps and the number of steps in the program. After each step, a snapshot is taken of the current position of all particles in 2 dimensions (x and y plane). These pistures are then transferred into a video to show the full particle simulation as shown in the video below. This project was done with Matthew Shortt as a 2 person team. We also added MPI for parallelized computations for greater speedups and efficiency.

4. Password Generator in Chrome Extension

  This project consists of a chrome extension that can be used to provide the user with a safe unique password for each website that they have a login for. It uses a combination of a salt file that is unique to the user, a user key, and the current domain name to generate the user a safe and secure password. The password is not stored anywhere meaning that it cannot be hacked. The output is hashed using SHA-256 to increase the difficulty of the password. Current market solutions involde password managers (storing passwords) and password generators (easier passwords to crack). This project was designed to incorporate the best solutions of both and add additional security for safe password protection. There is also a preferences page to choose certain characteristics that the user may want to see in a password.

5. Music Recommendation Application

  The goal of this project was to utilize a very large dataset and base the application off of what data was present in the dataset. Our group chose to do a million song dataset with a Java/Python implementation. We allowed the user to listen to a portion of a song. The user proceeded to answer a series of questions about what they liked and disliked about the song. Using those answers we compared the values to the data in the dataset to select another song that the user may prefer. We asked questions about tempo, artists, loudness, genre, and a few more. Comparing the results we were able to narrow down songs that the user may prefer to help them find new songs to add to their ipods/phones.
